### 5. Plano de limpeza recomendado (ordem exata)

1. Criar branch: git checkout -b chore/limpeza-repo
2. Remover arquivo vazio e commitar:
   - git rm backend/docker-compose.test.yml
   - git commit -m "chore: remove backend empty docker-compose.test.yml"
3. Mover fixtures e postman para pastas organizadas:
   - mkdir -p backend/tests/fixtures docs/postman
   - git mv AUTH_CADASTRO_*.json backend/tests/fixtures/
   - git mv backend/postman_collection.json docs/postman/
   - git commit -m "chore(tests/docs): mover fixtures e postman para locais organizados"
4. Arquivar documentos antigos:
   - mkdir -p docs/archive
   - git mv README_OLD.md docs/archive/README_OLD.md
   - git mv RELATORIO_COMPLETO.md docs/archive/RELATORIO_COMPLETO.md
   - git commit -m "chore(docs): archive old reports"
5. Revisar package.json raiz: confirmar uso; se não necessário, git rm package.json (revisão manual obrigatória).
6. Atualizar backend/.env.test.example para incluir JWT_SECRET e JWT_EXPIRES_IN; commitar.
7. Rodar validação local:
   - npm --prefix backend ci
   - cd backend && npm test (ou npm run test:with-db com docker)
8. Abrir PR com checklist e descrever arquivos arquivados/remoções.

Mudanças que exigem cuidado:
- Remoção do package.json na raiz — verificar CI/pipelines.
- Remover documentação crítica sem arquivar primeiro.
