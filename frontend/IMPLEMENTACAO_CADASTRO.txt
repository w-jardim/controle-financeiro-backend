====================================
INTEGRAÇÃO COM BACKEND
====================================

Endpoint: POST /api/auth/cadastro

Request Body:
{
  "nomeResponsavel": "string",
  "email": "string",
  "senha": "string",
  "nomeConta": "string"
}

Response (sucesso):
{
  "dados": {
    "mensagem": "Conta criada com sucesso"
  }
}

Response (erro 400):
{
  "mensagem": "Email já cadastrado" | "Dados inválidos"
}

Fluxo Implementado:
1. Usuário preenche formulário de cadastro
2. Frontend valida com Zod
3. Envia POST /api/auth/cadastro
4. Backend cria conta e usuário
5. Frontend faz login automático
6. Redireciona para dashboard

====================================
CÓDIGO IMPLEMENTADO
====================================

1. SCHEMA (register.schema.ts)
-------------------------------
export const registerSchema = z.object({
  nomeResponsavel: z.string().min(1, 'Nome do responsável é obrigatório'),
  email: z.string().email('Email inválido'),
  senha: z.string().min(6, 'Senha deve ter pelo menos 6 caracteres'),
  nomeConta: z.string().min(1, 'Nome da conta é obrigatório'),
});

2. SERVICE (authService.ts)
----------------------------
export async function registerApi(data: RegisterData): Promise<void> {
  await api.post<RegisterResponse>('/auth/cadastro', data);
}

Types:
- RegisterData: { nomeResponsavel, email, senha, nomeConta }
- RegisterResponse: { dados: { mensagem } }

3. PÁGINA (Register.tsx)
-------------------------
Componente funcional com:
- useForm<RegisterSchema> com zodResolver
- useState para loading e errorMsg
- useAuth para login automático após cadastro
- 4 inputs (nomeResponsavel, email, senha, nomeConta)
- Botão submit com estado "Criando conta..." / "Criar Conta"
- Link para /login
- Tratamento de erro da API

Fluxo onSubmit:
- Chama registerApi(data)
- Se sucesso: chama login(email, senha) → redirect automático
- Se erro: exibe mensagem de erro
