4. AUTENTICAÇÃO IMPLEMENTADA
==============================================================

4.1. FLUXO DE AUTENTICAÇÃO
--------------------------
1. Usuário acessa /login
2. Preenche email e senha
3. Frontend valida com Zod
4. Envia POST /api/auth/login
5. Backend retorna { dados: { mensagem, token } }
6. Token armazenado em localStorage
7. Redirecionamento para dashboard
8. Token enviado automaticamente em requisições (Authorization: Bearer)
9. 401 limpa sessão e redireciona para /login

4.2. COMPONENTES DE AUTENTICAÇÃO
--------------------------------

AuthContext (src/contexts/AuthContext.tsx):
- Estado: user, token, isAuthenticated, isLoading
- Métodos: login(email, senha), logout()
- Bootstrap: restaura token do localStorage ao carregar
- Integração: registra callback de logout no axios

AuthService (src/services/api/authService.ts):
- loginApi(email, senha): chama POST /auth/login
- Retorna token extraído de response.data.dados.token
- Tipagem forte com TypeScript

Axios Client (src/services/api/axios.ts):
- BaseURL: VITE_API_URL
- Interceptor Request: adiciona Authorization Bearer automaticamente
- Interceptor Response: trata 401 limpando token e chamando logout callback
- Export setLogoutCallback para integração com AuthContext

Storage (src/utils/storage.ts):
- setToken(token): salva em localStorage
- getToken(): recupera de localStorage
- clearToken(): remove de localStorage
- Key: 'cf_token'

ProtectedRoute (src/components/ProtectedRoute.tsx):
- Verifica isLoading (exibe "Carregando...")
- Verifica isAuthenticated
- Bloqueia acesso se não autenticado
- Redireciona para /login

Login Page (src/pages/auth/Login.tsx):
- Formulário com React Hook Form
- Validação Zod: email válido, senha ≥6 caracteres
- Estado loading durante requisição
- Exibição de erros de validação
- Exibição de erros da API
- Campos desabilitados durante loading
- Botão com feedback visual (Entrando...)

12. INTEGRAÇÃO COM BACKEND
==============================================================

12.1. ENDPOINT INTEGRADO
------------------------
POST /api/auth/login

Request:
{
  "email": "string",
  "senha": "string"
}

Response (sucesso):
{
  "dados": {
    "mensagem": "Login realizado com sucesso",
    "token": "jwt-token-string"
  }
}

Response (erro 401):
{
  "mensagem": "Email ou senha incorretos"
}

Response (erro 403):
{
  "mensagem": "Usuário inativo" | "Usuário sem conta"
}

12.2. CONTRATO VALIDADO
-----------------------
- Backend retorna estrutura { dados: { ... } }
- Frontend extrai response.data.dados.token
- Mensagens de erro extraídas de error.response.data.mensagem
- Token JWT enviado em Authorization: Bearer <token>

12.3. FLUXO PONTA A PONTA
-------------------------
1. Frontend envia POST /api/auth/login com { email, senha }
2. Backend valida credenciais
3. Backend gera JWT com { sub: userId, accountId, role }
4. Backend retorna { dados: { mensagem, token } }
5. Frontend salva token em localStorage
6. Frontend redireciona para /
7. Todas requisições subsequentes incluem Authorization: Bearer <token>
8. Backend valida token em rotas protegidas
9. Se 401, frontend limpa token e redireciona para /login
