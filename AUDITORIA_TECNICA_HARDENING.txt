AUDITORIA TÉCNICA — ENDURECIMENTO BACKEND
Data: 2026-03-27

Resumo
------
Auditoria focada em banco de dados, docker/env, contratos de resposta e documentação.

1) Problemas encontrados
------------------------
- Índices/UNIQUEs faltantes:
  - `transacoes` não possui UNIQUE(account_id, tipo, descricao) (código assume duplicidade).
  - `alunos` código referencia `uq_alunos_nome_data` e `uq_alunos_nome_telefone` que não existem no SQL.

- Tratamento de erros inconsistente:
  - `errorHandler.js` intercepta `ER_DUP_ENTRY` e retorna mensagem fixa de transações, mascarando outros casos.

- Contratos inconsistentes:
  - `cts` e `transacoes` retornam objeto paginado; `alunos` retorna array simples — inconsistência API.

- Docker / env mismatch:
  - `docker-compose.test.yml` mapeia porta 3308; `backend/.env.test.example` tem DB_PORT=3307.
  - `backend/.env.test` (com segredos de teste) está versionado no repositório.

- Foreign keys / ON DELETE:
  - `transacoes.account_id` usa ON DELETE RESTRICT (revisar se CASCADE é desejado em fluxos de remoção).
  - `alunos.ct_id` é RESTRICT (impede remover CTs com alunos; pode ser correto ou exigir SET NULL/CASCADE).

2) Causa raiz
--------------
- Falta de garantias aplicadas no nível do banco (constraints) — validações aplicacionais sozinhas não protegem contra concorrência.
- Tratamento global de erros que padroniza mensagens sem mapear constraints específicas.
- Divergência entre ambiente docker e exemplos de env causa falhas intermitentes em CI/local.

3) Impacto técnico
------------------
- Risco de duplicidade de dados por condições de corrida.
- Mensagens de erro enganosas e dificuldades de depuração.
- Clientes precisam lidar com formatos de listagem diferentes.
- Testes e CI podem falhar por mismatch de porta/env.

4) Correções recomendadas (prioritárias)
------------------------------------
a) Banco (aplicar em homolog antes de produção):

-- Adicionar UNIQUEs esperados
ALTER TABLE transacoes
  ADD UNIQUE KEY uq_transacoes_account_tipo_descricao (account_id, tipo, descricao);

ALTER TABLE alunos
  ADD UNIQUE KEY uq_alunos_nome_data (account_id, nome, data_nascimento),
  ADD UNIQUE KEY uq_alunos_nome_telefone (account_id, nome, telefone);

Nota: verifique índices existentes e valores NULLs antes de aplicar (backups/migração).

b) Código
- Ajustar `src/shared/middlewares/errorHandler.js` para mapear `ER_DUP_ENTRY` por constraint (usar `erro.sqlMessage`) ou delegar mensagens para `AppError` lançado pelo service.

c) Docker / env
- Unificar porta: atualizar `backend/.env.test.example` para `DB_PORT=3308` (ou mapear compose para 3307). Manter escolha consistente.
- Remover `backend/.env.test` do repositório (`git rm --cached backend/.env.test`) e adicionar instrução em README para copiar do `.env.test.example`.

d) API
- Padronizar listagens: escolher padrão paginado `{ pagina, limite, total, totalPaginas, dados }`. Atualizar `alunoController` para adaptar (ou documentar exceção).

5) O que é prioritário

3. Unificar porta env/compose e remover secrets do VCS.
